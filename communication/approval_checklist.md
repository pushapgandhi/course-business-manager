# Communication Approval Checklist

Use this checklist before any message is sent.

## Message Review

| Check | Pass/Needs Edit |
|---|---|
| Recipient or audience is correct |  |
| Channel is correct |  |
| Purpose is clear |  |
| Tone matches the relationship |  |
| Facts are accurate |  |
| CTA is clear and singular |  |
| Links are correct |  |
| Attachments are included or intentionally omitted |  |
| Timing or deadline is correct |  |
| No sensitive information is exposed |  |
| Message is approved by the user |  |

## Send Approval

Before sending, confirm:

```text
I approve this exact message for this recipient, channel, and sending method.
```

Do not treat vague confirmations like "looks good" as approval to send unless the user also asked for sending in the same context. If unclear, ask for explicit send approval.

## Post-Send Record

After sending or preparing a manual-send message, record:

- date
- recipient or audience
- channel
- subject or message type
- CTA
- status: sent, copy-ready, waiting, or revised
- follow-up date, if any
