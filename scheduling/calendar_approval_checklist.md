# Calendar Approval Checklist

Use this checklist before creating, updating, rescheduling, or canceling a calendar event.

## Event Review

| Check | Pass/Needs Edit |
|---|---|
| Action type is correct |  |
| Event title is clear |  |
| Attendees are correct |  |
| Attendee emails are confirmed when invites are needed |  |
| Date and time are correct |  |
| Timezone is correct |  |
| Duration is correct |  |
| Location or meeting link is correct |  |
| Agenda or notes are accurate |  |
| Reminder settings are correct |  |
| Conflict check is complete when a calendar tool is available |  |
| Separate recipient email is needed for scheduled or rescheduled meetings |  |
| Recipient email includes title, date/time/timezone, meeting link or location, and agenda |  |
| User approved the exact calendar action |  |

## Approval Phrase

Before changing the calendar, confirm:

```text
I approve this exact calendar action with these event details.
```

If the user says "looks good" or gives a vague confirmation, ask whether that approval includes changing the calendar.

## Post-Update Record

After updating the calendar, record:

- date of action
- action type
- event title
- attendees
- final date/time/timezone
- location or link
- notification status
- recipient email status
- next follow-up, if any
