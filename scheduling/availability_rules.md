# Availability And Conflict Rules

Use these rules when checking availability or proposing meeting times.

## Defaults

- Default timezone: user's known timezone.
- Default duration: 30 minutes when the user does not specify.
- Default reminder: 15 minutes before the meeting.
- Default number of proposed slots: three.
- Do not schedule outside the user's stated availability window.

## Conflict Handling

When a calendar tool is available:

1. Check the requested time range.
2. Identify conflicts with existing events.
3. Treat busy events as unavailable unless the user says otherwise.
4. Propose alternatives when the requested time is unavailable.
5. State whether the proposed time is free, busy, tentative, or unknown.

When a calendar tool is not available:

1. Say that availability cannot be verified from the calendar.
2. Use only the user-provided availability.
3. Ask for confirmation before preparing event details.

## Time Slot Quality

Prefer slots that:

- fit the requested date range
- avoid existing conflicts
- provide enough meeting duration
- avoid very short gaps when possible
- leave preparation time before important meetings when known

## Approval Boundary

Availability checks and proposed slots are safe to provide without approval.

Creating, updating, rescheduling, canceling, or adding reminders requires explicit approval.
